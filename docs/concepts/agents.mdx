# Compliance Agents

ZebraTruth uses a pipeline of specialized AI agents, each focused on a specific compliance domain. Agents are added regularly — always use `GET /agents` to discover available agents dynamically.

## Current Agents

| Agent | `agentId` | Phase | Input | What It Checks |
|-------|-----------|-------|-------|----------------|
| **AI Laws** | `ai-laws` | 1 | Text | AI-content laws — disclosure, watermarking, transparency (EU AI Act Art. 50, California SB 942, Deep Synthesis Provisions, IT Rules, ELVIS Act, etc.) |
| **Advertising Law** | `advertising-law` | 2 | Text | Claim substantiation, endorsement rules, comparative advertising, targeting restrictions |
| **Platform Policy** | `platform-policy` | 2 | Text | YouTube, Instagram, Facebook, TikTok, LinkedIn, Douyin, Weibo policies |
| **Metadata & Labeling** | `metadata-labeling` | 3 | Text | C2PA Content Credentials, AI disclosure labels, machine-readable provenance |
| **Rights Clearance (Text)** | `rights-clearance-text` | 3 | Text | Celebrity references, trademarks, copyrighted characters, music licensing, voice cloning |
| **Rights Clearance (Image)** | `rights-clearance-image` | 3 | Image | Restricted categories (alcohol, weapons, tobacco, gambling, pharmaceuticals, financial products), content safety (adult / violence / racy), brand logos, stock-agency watermarks, C2PA metadata. See the [Restricted Categories guide](/guides/restricted-categories). |
| **Video Compliance Check** | `compliance-check-video` | async | Video | Restricted categories, content safety, on-screen text (burned-in disclaimers, `#ad` labels), spoken-audio claims, brand logos, watermarks, AI provenance — orchestrated async pipeline via `POST /v1/compliance/check-video`. Findings carry timestamped `evidenceIds` and `timelineLocation` so clients can jump to the offending second. |
| **Compliance Score** | `compliance-score` | 4 | All | Deterministic scoring from all checks |

## Pipeline Phases

In **full mode**, agents run in 4 sequential phases with content enrichment:

1. **Phase 1** — AI Laws agent runs first; produces the `ComplianceProfile` consumed by downstream agents
2. **Phase 2** — Advertising Law + Platform Policy agents (parallel), enriched with Phase 1 findings
3. **Phase 3** — Metadata + Rights agents (parallel), enriched with Phase 1+2 findings
4. **Phase 4** — Score agent aggregates all checks

In **fast mode**, all agents run in parallel without enrichment.

## 3-Stage Sub-Agent Pipeline

Each text agent internally runs a 3-stage pipeline. The stages are semantic roles
rather than specific providers — the underlying models may change over time, and
that's fine: the `stackVersion` field in every response captures the current
stack identity opaquely, so clients can detect changes without knowing what's
inside.

| Stage | Role | Source |
|-------|------|--------|
| A | Legal research & citation retrieval | Tenant-scoped [legal corpus](/concepts/legal-corpus) with stale-while-revalidate refresh |
| B | Content-specific risk interpretation | Fresh per request |
| C | Structured compliance composition + classification | Fresh per request |

Stage A is sourced from a tenant-scoped legal corpus, refreshed out-of-band by a
background worker. Reads return immediately for fresh rows, return immediately while
triggering a background refresh for stale-but-acceptable rows, and block on a
synchronous fetch only when missing or expired. Stages B and C depend on the actual
content being checked, so they always run fresh.

See the [Legal Corpus concept page](/concepts/legal-corpus) for the freshness model
and manual refresh.

## Jurisdiction and Platform Scoping

Agents don't reason about "all the law in the world" — each invocation is constrained
to the specific jurisdictions and platforms the request asks for. This has two effects:

- **Law injection is dynamic.** Each agent pulls only the relevant slice of the
  [jurisdiction law map](/concepts/jurisdiction-law-map) into its prompt. A request against
  `["us"]` sees only US law; a request against all 5 jurisdictions sees all 5.
- **Findings stay in-scope.** An agent given only US law won't volunteer EU-specific
  issues. Every finding traces back to a law or policy that was in scope for this
  request. On image-rights findings, this also shows up as per-jurisdiction
  [citations](/guides/image-rights-clearance#per-jurisdiction-citations).

The set of permitted jurisdictions and platforms is further bounded by your tenant's
subscription — see [Tenant Onboarding and Scoping](/guides/tenant-onboarding-and-scoping).
Requesting something outside your subscription returns 403 before any agent runs.

## Invoking a Single Agent

Call any agent directly via `POST /agents/{agentId}`. Costs ~8-12 credits vs ~47 for a full check.

See [Individual Agents guide](/guides/individual-agents) for details.
